## 前提
構想フェイズは完了し、仕様フェイズに移行する。
構想レビューでは、デモとしての伝達力を高めるために、クラスタリング前後のビフォー・アフターの見せ方を仕様で検討することが有効である、という示唆が得られている。

本ドキュメントは、仕様フェイズ開始にあたり、現時点で合意した事項と、仕様検討の依頼内容を整理するものである。

## 2. 技術スタック方針

以下の方針で仕様化を進めたい。

- SwiftUI による macOS アプリを開発対象とする
- ローカル実行を前提とし、Web サーバーは準備しない
- CSV ファイルは macOS 標準の SQLite に取り込み、元データ・埋め込み・クラスタリング結果・トピック生成結果をテーブルとして保存する
- OpenAI API のみ対応する
- 並列処理は Swift の Structured Concurrency を前提とする
- クラスタリング計算では Apple プラットフォームで利用可能な高速計算基盤を活用する前提でよいが、具体的な API 選定は仕様として過度に固定しすぎず、設計で確定してもよい

---

## 3. データ仕様

### 3.1 入力データ
- 入力は CSV ファイル
- サンプルデータ位置:
  - `SDD Demo/Data/文字起こし結果_masked.csv`

### 3.2 CSV対応要件
- RFC 4180 形式に対応する
- 文字コード判定に注意する
  - BOM 有無
  - Shift_JIS 判定ミスの可能性
- 改行コードに注意する
  - CRLF
  - CR 単独
  - LF 単独

### 3.3 永続化対象
SQLite には少なくとも以下を保存対象とすること。

- 読み込んだ元データ
- 埋め込み結果
- クラスタリング結果
- トピック生成結果
- 利用したパラメータの履歴

---

## 4. 分析処理仕様

処理フローは以下を基本とする。

1. CSV を読み込む
2. 埋め込み対象列を選択する
3. 選択列に対して埋め込みを生成する
4. 算出済み埋め込みに対してクラスタリングを実行する
5. 各クラスターから代表候補を抽出する
6. データセット全体の文脈を踏まえて LLM によりトピックを生成する
7. クラスター件数・代表内容・可視化結果を表示する

### 4.1 埋め込み
- メインウィンドウに表示中のデータセットから列選択できること
- 設定画面で選択した埋め込みモデルを利用すること

### 4.2 クラスタリング
- k-means++
- DBSCAN

各手法について、必要なパラメータを設定可能とすること。

### 4.3 トピック生成
- 保存済みクラスタリング結果を選択して実行できること
- 各クラスターの重心または平均に近い要素を数件選ぶこと
- クラスター同士のトピック重複を抑えるため、元データからランダムに数件抽出し、データセット全体が何を表しているかを LLM に補助的に与えること
- 生成したトピックは再利用可能な形で保存すること

## 5. 画面仕様

### 5.1 メインウィンドウ
- 読み込んだ CSV データを表形式で表示する

### 5.2 アプリメニュー > 設定
設定画面では以下を扱えること。

- OpenAI API キー入力
- LLM モデル選択
- 埋め込みモデル選択
- API キー有効性確認

### 5.3 ファイルメニュー > 開く
- CSV ファイル選択ダイアログを表示する
- 選択ファイルを読み込み、メインウィンドウへ表示する

### 5.4 ファイルメニュー > 分析
分析ダイアログに以下を持つこと。

#### 埋め込みタブ
- データセット列一覧表示
- 埋め込み対象列選択
- 埋め込み実行

#### クラスタリングタブ
- 算出済み埋め込み一覧表示
- クラスタリング手法選択
- 手法ごとのオプション設定
- クラスタリング実行
- 保存済みクラスタリング選択
- トピック生成実行

### 5.5 結果表示
- 生成済みトピックを別画面の表で確認できること
- クラスター件数を確認できること
- クラスター選択から可視化画面へ遷移できること

## 6. デモ表示仕様

デモの伝達力を高めるため、以下を仕様検討対象に含めてほしい。

- クラスタリング前の生データの見え方
- クラスタリング後の整理結果の見え方
- 両者のビフォー・アフター比較
- クラスターごとの代表トピック表示
- 可視化結果が「面白い」だけでなく「意味が伝わる」こと

3D散布図は、参加者の興味喚起のための有力候補として扱ってよい。
ただし、可視化は主役ではなく、クラスタリング結果の理解を補助するものとして位置づけること。

## 7. CTO補佐からの仕様上の論点

以下は、仕様レビュー時に特に見てほしい論点である。

- ローカル実行で完結することを、どの仕様で担保するか
- 埋め込み未実行・クラスタリング未実行・APIキー未設定などの状態遷移をどう整理するか
- 長文レコードが存在する前提で、埋め込み対象の前処理方針を仕様上どこまで定めるか
- トピック生成の再現性と説明可能性をどの程度担保するか
- 3D可視化が未対応環境でも、デモ価値を損なわない代替表示をどう扱うか
