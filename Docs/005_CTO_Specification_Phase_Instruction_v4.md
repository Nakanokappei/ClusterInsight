## 1. 合意事項

### 1.1 ローカル実行・配布
- 成果物は macOS アプリ（`.app`）とする
- `xcodebuild` + `xcodegen` で生成可能であること
- Developer ID Application 署名 + Apple 公証（Notarization）+ Staple を実施すること
- 社内ネットワークの掲示板経由で配布すること（zip形式）
- 各自の Mac 上で Gatekeeper 警告なしに実行できることを要件とする
- 【v4変更】Hardened Runtime + セキュアタイムスタンプ必須を明記

### 1.2 状態遷移
- 埋め込み未実行・クラスタリング未実行状態は、保存済みリストの有無で判別する
- API キー未設定時は、関連操作を無効化してユーザーへフィードバックする
- 【v4変更】API キーは macOS Keychain に暗号化保存する（UserDefaults 不可）

### 1.3 長文レコードの扱い
- 埋め込み入力は 8K トークン相当を上限とする
- 日本語は暫定的に「1文字 = 1トークン」で近似する（簡便な近似であり、実トークン数はこれより多くなりうる）
- 上限を超過しそうな場合は後方を切り捨てる
- 本件はデモ用途であり、厳密な意味保持よりも単純性・再現容易性を優先する

### 1.4 トピック生成
- 【v4変更】データセットのランダムサンプルからLLMにデータセットの正体を推定させ、その結果でグラウンディングする
- 隣接トピックの例示により、差異が際立つようシステムプロンプトで誘導する
- 【v4変更】トピック名の重複検出を行い、重複時は差別化プロンプトで再生成する（最大2回）
- 【v4変更】大きいクラスターから順に処理し、大クラスターが先にトピック名を確定する
- 【v4変更】プロンプトで抽象的な表現を避け、データ中の具体的キーワードを含めるよう指示する
- デモ用途のため、高度な再現性・厳密性は必須要件としない

### 1.5 3D可視化
- 対応OSは macOS Tahoe 26 以上を前提とする
- 【v4変更】3D座標生成方式は PCA + クラスター重心スケーリングを用いる（簡易UMAP は品質不足により不採用）
- 未対応環境への代替表示は今回のスコープ外とする

### 1.6 UIアーキテクチャ【v4新規追加】
- メインウィンドウ: データテーブル表示のみ（サイドバーなし）
- 分析ウィンドウ: タブ切り替え（埋め込み / クラスタリング / 3D散布図）
- 操作はメニューバー駆動（ファイル → CSV を開く、分析 → 分析ウィンドウ）
- 分析ウィンドウは起動時に非表示、明示的に開いたときのみ表示

### 1.7 埋め込み処理【v4新規追加】
- 埋め込み対象列はユーザーが選択可能とする
- 並列処理はAPIレート制限ヘッダー（x-ratelimit-remaining-requests, x-ratelimit-reset-requests）を読み取り、適応的にバッチサイズを調整する

## 2. 仕様で固定する事項と設計へ委ねる事項

### 仕様で固定（変更にはCTO判断が必要）
- 成果物形態: macOS署名済み・公証済みアプリ（.app）
- 対応OS: macOS Tahoe 26以上
- 処理フロー: CSV → 埋め込み → クラスタリング → トピック生成 → 表示
- トークン上限: 8K（1文字≒1トークン近似、後方切り捨て）
- デモ優先の割り切り（厳密な再現性は必須要件としない）
- UIはメニューバー駆動＋2ウィンドウ（データ＋分析タブ）

### 設計に委任（エンジニア判断で決定可能）
- 具体的なAPI選定（埋め込みモデル、LLMモデル）
- クラスタリングアルゴリズムの選定
- PCA + 重心スケーリングのパラメータ調整
- 代表データ抽出数・ランダム抽出数の具体値
- トピック生成プロンプトの文面
- UIレイアウト・操作フローの詳細
