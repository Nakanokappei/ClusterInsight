## 4. 3D座標生成方式

### 【v2変更】PCA + クラスター重心スケーリング

v1: 簡易UMAP（方式D）→ **v2: PCA + クラスター重心スケーリング**

#### アルゴリズム
1. **グローバルPCA** — 全埋め込みベクトルの Gram 行列を LAPACK ssyev_ で固有値分解、上位3固有ベクトルに射影
2. **クラスター重心算出** — 各クラスターの3D平均座標を計算
3. **重心スケーリング** — 重心間の最近傍距離に基づき、視覚的に分離されるよう外側に拡大（上限1.8倍）
4. **メンバー配置** — 各メンバーを重心周辺に配置、95パーセンタイル距離でクラスター半径を正規化
5. **正規化** — [-1, 1] 立方体に正規化してSceneKitに渡す

#### パラメータ
| パラメータ | 値 | 説明 |
|-----------|-----|------|
| targetMinNN | max(0.2, 0.5 / ∛K) | 重心間の目標最小距離 |
| scaleFactor上限 | 1.8 | PCA配置からの最大拡大率 |
| memberRadius | 0.3 * √(n/N) | クラスターの表示半径 |

#### 不採用理由（簡易UMAP）
OpenAI text-embedding-3-small のコサイン距離分布が 0.05〜0.3 に密集しており、簡易SGDベースのUMAPではクラスター分離を達成できなかった。3回の書き直しを経て PCA ベースに切り替え。

## 5. トピック生成フロー設計

### 【v2変更】3段階のトピック生成

#### 5.1 Phase 1: データセット特徴推定
- 20件のランダムサンプルをLLMに提示
- 「このデータセットは何のデータか」を100文字以内で推定させる
- 推定結果をグラウンディングテキストに含める

#### 5.2 Phase 2: トピック生成（クラスターサイズ降順）
- 大きいクラスターから順に処理（先にトピック名を確定する権利）
- 重心近傍5件を代表データとして抽出
- システムプロンプト: データセット特徴 + サンプルデータ + 既生成トピック一覧
- ユーザープロンプト: 代表データ + 「具体的キーワードを含めよ」指示 + 良い例・悪い例

#### 5.3 Phase 3: 重複検出 + 再生成
- 生成後にトピック名を既存トピック名と比較
- 重複発見時: 重複元クラスターの代表データ3件も提示し「この2つの違いに注目して別の名前を付けよ」と指示
- 最大2回再生成
