## 前提
実装フェイズは完了した。
本ドキュメントでは、検証・デモフェイズへの移行を指示する。

---

## 1. 実装評価

実装報告を確認した。

- BUILD SUCCEEDED
- 全マイルストーン達成
- データパイプライン一貫動作確認済み

→ **実装は完了と認める**

---

## 2. フェイズ移行

以下の通り移行する：

実装 → 検証 → デモ

---

## 3. 検証タスク

### V1: 正常系検証
- CSV読込 → 埋め込み → クラスタリング → UMAP → トピック生成
- 一連の処理がエラーなく完走すること

### V2: 境界条件
- 空レコードのスキップ確認
- 長文切り捨て確認
- APIキー未設定時のUI制御確認

### V3: データ整合性
- 再実行時のデータ破棄動作確認
- embedding / clustering / topics の整合確認

### V4: パフォーマンス
- 500件での実行時間
- メモリ使用量

---

## 4. デモ準備

### D1: デモシナリオ
1. CSV読込（生データ提示）
2. 埋め込み実行（ブラックボックス化）
3. クラスタリング実行
4. 3D表示（驚きポイント）
5. トピック表示（理解ポイント）

### D2: 強調ポイント
- ローカルPCで動作
- 自動でグルーピング
- 意味的に理解できる出力

---

## 5. CTO補佐指摘

必ず確認すること：

- 3Dが「面白い」で終わっていないか
- トピックが「意味として納得できるか」
- 操作が詰まらないか（デモ時間短縮）
